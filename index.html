MVP Specification

Music Flow

An interactive mobile and web audio-visual experience designed to guide users into a calm flow state through rhythmic interaction and reactive tranquil art.

View Product Flow Read Full PRD

Operational Pipeline

The MVP proves the full path from uploaded audio to generated gameplay data and synchronized visual rendering.

01

Audio Upload

Admins upload ambient `.mp3` or `.wav` tracks to Supabase Storage.

02

Audio Analysis

Edge Functions extract beats, transients, and structural sections.

03

LLM Choreography

Gemini interprets composition structure and produces a sequenced level map.

04

Reactive Playback

Flutter renders synchronized hit nodes and fluid shader visuals.

Recommended Stack

  • UI
    Flutter 3.x and WebAssembly
  • Graphics
    Flutter Canvas, Impeller, and GLSL
  • Backend
    Supabase Postgres and Edge Functions
  • Storage
    Supabase Storage CDN buckets
  • Analysis
    Essentia.js compiled to WASM
  • Choreography
    Gemini Flash API for JSON mapping

Data Shape

Tracks persist streamable audio metadata alongside a generated rhythmic timeline. 

{
  "theme": "aurora_fractal",
  "events": [
    { "time_ms": 1200, "action": "spawn_node", "lane": 2 },
    { "time_ms": 4800, "action": "shift_visual_state" }
  ]
}

MVP Scope

  • Manual admin upload interface for processing new audio tracks.
  • Backend pipeline that produces valid `level_map` JSON.
  • Minimal frontend with descending hit nodes over a fluid canvas.
  • Flutter Wasm browser rendering verification.
  • No complex profiles, creator tools, leaderboards, or client-side multi-track mixing.
  • Haptic feedback is reserved for post-MVP mobile refinement.
